# Implementation Plan: Corporate Contribution Lens

**Branch**: `493-feat-corporate-contribution-lens-for-rep` | **Date**: 2026-04-29 | **Spec**: [spec.md](spec.md)

## Summary

Add a single "Company" filter to the Contributors tab that, without re-running analysis, shows per-repo corporate commit counts, corporate author counts, and corporate %, plus a summary row across all repos. Attribution uses two complementary signals already derivable from existing per-window commit data: GitHub org membership (existing pipeline extended to capture per-org author keys) and commit email domain (new per-domain tracking for email-based actors).

## Implementation phases

### Layer 1 — Type extension (analysis-result.ts)

Add three optional fields to `ContributorWindowMetrics`:

```typescript
commitAuthorsByExperimentalOrg?: Record<string, string[]> | Unavailable
commitCountsByEmailDomain?:       Record<string, number>   | Unavailable
commitAuthorsByEmailDomain?:      Record<string, string[]> | Unavailable
```

Backwards-compatible: `?` means existing fixtures and serialized results require no changes.

Also update the `'unavailable'` initializers in `createUnavailableContributorWindowMetrics()` and the zero-window early-return in `buildExperimentalMetricsByWindow` to include the new fields.

### Layer 2 — Analyzer extension (analyze.ts)

Extend `buildExperimentalMetricsByWindow` to populate the three new fields alongside the existing `commitCountsByExperimentalOrg`:

**Org signal extension** (inside the login-based attribution loop):
```typescript
const commitAuthorsByExperimentalOrg = new Map<string, Set<string>>()
// per org, add actorKey to the set:
const authorSet = commitAuthorsByExperimentalOrg.get(org) ?? new Set<string>()
if (authorSet.size < 500) authorSet.add(actorKey)
commitAuthorsByExperimentalOrg.set(org, authorSet)
```

**Email signal** (new branch for email-based actors — where `login` is absent but `email` is present):
```typescript
const commitCountsByEmailDomain = new Map<string, number>()
const commitAuthorsByEmailDomain = new Map<string, Set<string>>()
// For nodes where getCommitActorKey returns 'email:...'
const email = node.author?.email?.toLowerCase()
if (email) {
  const domain = email.split('@')[1]
  if (domain) {
    commitCountsByEmailDomain.set(domain, (commitCountsByEmailDomain.get(domain) ?? 0) + 1)
    const dSet = commitAuthorsByEmailDomain.get(domain) ?? new Set<string>()
    if (dSet.size < 500) dSet.add(`email:${email}`)
    commitAuthorsByEmailDomain.set(domain, dSet)
  }
}
```

Return these alongside the existing fields.

### Layer 4 — UI component (CorporateContributionPanel)

- Single `<input type="text">` labelled "Company" with placeholder `e.g. microsoft`
- When empty: renders nothing (just the empty input).
- When non-empty: renders the per-repo table + summary row.
- Experimental boundary: wrapped in a bordered section with disclosure text covering the three known limitations (FR-013).
- Uses `computeCorporateMetrics` from Layer 3.
- No local window state — receives `windowDays` as a prop from `ContributorsView`.

Add `CorporateContributionPanel` at the top of `ContributorsView`, passing `results` and the existing `windowDays` state. No new state in `RepoInputClient`.
