# Implementation Plan: Foundation Input Mode

**Branch**: `413-feat-foundation-aware-input-mode-as-a-to` | **Date**: 2026-04-23 | **Spec**: [spec.md](./spec.md)

## Summary

Add a third top-level input mode ("Foundation") alongside Repositories and Organization. Foundation mode accepts repos or an org slug in a single smart input field, detects the kind automatically, and renders either per-repo CNCF Sandbox readiness reports or an org-level candidacy ranking. Simultaneously removes foundation elements (dropdown, CNCF Readiness tab, CNCF Candidacy org tab) from Repositories and Organization modes so all foundation work is consolidated in one place. No new API endpoints are required.

### Phase 2 ‚Äî Smart input parser (TDD first)

**File**: `lib/foundation/parse-foundation-input.ts`

1. Write `parse-foundation-input.test.ts` covering:
   - `owner/repo` ‚Üí `{ kind: 'repos', repos: ['owner/repo'] }`
   - Multiple repos (newline, comma, space separated) ‚Üí `{ kind: 'repos', repos: [...] }`
   - Full GitHub URL `https://github.com/owner/repo` ‚Üí repos
   - Bare slug `cncf` ‚Üí `{ kind: 'org', org: 'cncf' }`
   - `github.com/cncf` ‚Üí `{ kind: 'org', org: 'cncf' }`
   - `https://github.com/cncf` ‚Üí `{ kind: 'org', org: 'cncf' }`
   - `https://github.com/orgs/cncf/projects/14` ‚Üí `{ kind: 'projects-board', url: '...' }`
   - Empty string ‚Üí `{ kind: 'invalid', error: '...' }`
   - Unrecognised pattern ‚Üí `{ kind: 'invalid', error: '...' }`
2. Implement `parseFoundationInput(input: string): FoundationParseResult` reusing `parseRepos` and `normalizeOrgInput` internally.

## Risk & Mitigations

| Risk | Mitigation |
|---|---|
| `ResultsShell` removing `aspirantResult` breaks existing CNCF Readiness display | Foundation mode is the only remaining consumer; Repositories mode no longer sets `aspirantResult` on the response path |
| Extended `FoundationTarget` union causes TS exhaustive-check errors | Fix all switch statements in Phase 1 before other phases begin |
| Shareable URL `?repos=` param conflicts with new `?mode=foundation` | Params are orthogonal ‚Äî `decodeRepos` checks `repos` key, `decodeFoundationUrl` checks `mode=foundation` |
| Nudge adds visual noise to Org/Repos results | `FoundationNudge` is intentionally minimal (single-line callout); dismiss is not required per spec |
