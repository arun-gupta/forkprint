# Implementation Plan: Dev-only server-side PAT fallback

**Branch**: `207-dev-only-server-side-pat-fallback-to-unb` | **Date**: 2026-04-14 | **Spec**: [spec.md](./spec.md)

## Summary

Add a narrow, dev-only fallback that lets server-side code use `process.env.DEV_GITHUB_PAT` for GitHub API calls when `NODE_ENV === 'development'`. Bypass the OAuth sign-in on the client via a server-gated `/api/auth/dev-session` probe. Enforce loud failure in production via a boot-time assertion.

## Technical Context

- **Language/Version**: TypeScript 5.x (Next.js 16+ App Router)
- **Primary Dependencies**: None new. Existing Next.js instrumentation hook (`instrumentation.ts`) and route handlers.
- **Storage**: N/A (stateless)
- **Testing**: Vitest + React Testing Library for unit; existing OAuth e2e unchanged.
- **Target Platform**: Next.js server runtime (nodejs), browser for `AuthGate` only.
- **Project Type**: Web application (Next.js monolith).

## Phase 1 — Design

### `lib/auth/dev-pat.ts`

```ts
export function isDevPatActive(): boolean {
  return process.env.NODE_ENV === 'development'
    && typeof process.env.DEV_GITHUB_PAT === 'string'
    && process.env.DEV_GITHUB_PAT.length > 0
}

export function resolveServerToken(clientToken: string | null | undefined): string | null {
  if (isDevPatActive()) return process.env.DEV_GITHUB_PAT!
  return clientToken ?? null
}

export function assertDevPatNotInProduction(): void {
  if (process.env.NODE_ENV === 'production' && process.env.DEV_GITHUB_PAT) {
    throw new Error(
      'DEV_GITHUB_PAT is set but NODE_ENV=production. ' +
      'This is a dev-only variable. Remove it from production env or set NODE_ENV=development.'
    )
  }
}
```

### Route handler changes

- `app/api/analyze/route.ts` and `app/api/analyze-org/route.ts`: replace the `const token = body.token; if (!token) 401` block with `const token = resolveServerToken(body.token); if (!token) 401`. No other logic changes.
- `app/api/auth/dev-session/route.ts`: returns `{ enabled: isDevPatActive(), username: 'dev' }` (username only when enabled).

### `AuthGate.tsx` change

On mount, `fetch('/api/auth/dev-session')`. If `enabled === true`, call `signIn({ token: 'dev-pat', username })`. This happens before the existing OAuth-fragment check and makes sign-in a no-op when dev-PAT is active. Failure or disabled → existing OAuth flow untouched.

### `instrumentation.ts` change

Add `await import('./lib/auth/dev-pat').then(m => m.assertDevPatNotInProduction())`. Runs on every server boot regardless of `NODE_ENV`; the assertion itself only throws under the prod+PAT misconfiguration.

## Risks & Mitigations

- **Risk**: Developer forgets to unset `DEV_GITHUB_PAT` before deploying. **Mitigation**: Startup assertion throws loudly in production.
- **Risk**: Sentinel `"dev-pat"` token leaks into a real GitHub request. **Mitigation**: The token server-side is resolved via `resolveServerToken`; the sentinel never becomes the server's resolved token because `isDevPatActive` is checked first.
- **Risk**: `/api/auth/dev-session` accidentally leaks PAT. **Mitigation**: Handler only returns a boolean + literal username string; unit test asserts absence of token string.
